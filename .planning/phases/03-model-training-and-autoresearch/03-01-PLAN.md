```python
def always_home_baseline(df: pd.DataFrame) -> float:
```
- Filter to rows where home_win is not NaN (exclude ties)
- Return df["home_win"].mean() -- proportion of home wins IS the accuracy when predicting "home always wins"
- Must work on any season's feature DataFrame

```python
def better_record_baseline(df: pd.DataFrame) -> float:
```
- Filter to rows where home_win is not NaN (exclude ties)
- Filter to rows where home_rolling_win AND away_rolling_win are not NaN (need records to compare)
- Predict home team wins when home_rolling_win > away_rolling_win
- Predict away team wins when away_rolling_win > home_rolling_win
- When tied records (home_rolling_win == away_rolling_win), predict home team wins (home fallback)
- Return accuracy = (predictions == actuals).mean()

```python
def compute_baselines(df: pd.DataFrame, season: int) -> dict:
```
- Filter df to the given season
- Call always_home_baseline and better_record_baseline on the filtered data
- Return dict with keys: "season", "always_home_accuracy", "better_record_accuracy", "game_count" (number of non-tie games evaluated)
- Log the game_count so we can verify both baselines used the same set

All functions import TARGET from features.definitions and use it for column reference.
The module docstring must state: "Baseline computation. Never modify during autoresearch experiments (only train.py is modified)."
