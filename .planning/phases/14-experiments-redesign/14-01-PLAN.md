  <action>
    Rewrite ExperimentTable.tsx to fix two issues:

    **Issue 1 -- Column misalignment (EXPR-01):**
    The root cause is that the `Collapsible` component from `@base-ui/react` renders a `<div>` element as its root. This `<div>` wraps around the `<tr>` rows inside `<tbody>`, creating invalid HTML (`<tbody><div><tr>...</tr></div></tbody>`). Invalid nesting breaks the browser's table layout algorithm, causing column widths in data rows to not align with header row widths.

    Fix: Remove the `Collapsible`, `CollapsibleTrigger`, and `CollapsibleContent` wrappers entirely. The component already manages expand/collapse state via `expandedId` state variable. Replace with:
    - A plain `<TableRow>` with an `onClick` handler that toggles `expandedId` (same as current `onOpenChange` logic)
    - A conditionally rendered second `<TableRow>` (when `expandedId === exp.experiment_id`) containing the `<TableCell colSpan={7}>` with `<ExperimentDetail>`

    Remove these imports (no longer needed):
    ```
    import { Collapsible, CollapsibleContent, CollapsibleTrigger } from "@/components/ui/collapsible";
    ```

    The resulting JSX for each experiment in the sorted.map should be:
    ```tsx
    <Fragment key={exp.experiment_id}>
      <TableRow
        className="hover:bg-secondary/50 cursor-pointer transition-colors"
        onClick={() => setExpandedId(expandedId === exp.experiment_id ? null : exp.experiment_id)}
      >
        <TableCell className="font-mono text-xs">{exp.experiment_id}</TableCell>
        <TableCell className="whitespace-normal">{exp.hypothesis}</TableCell>
        <TableCell>{(exp.val_accuracy_2023 * 100).toFixed(1)}%</TableCell>
        <TableCell>{(exp.val_accuracy_2022 * 100).toFixed(1)}%</TableCell>
        <TableCell>{(exp.val_accuracy_2021 * 100).toFixed(1)}%</TableCell>
        <TableCell>{exp.log_loss.toFixed(4)}</TableCell>
        <TableCell>
          {exp.keep ? (
            <Badge className="bg-status-success/15 text-status-success border-0">Kept</Badge>
          ) : (
            <Badge className="bg-status-error/15 text-status-error border-0">Reverted</Badge>
          )}
        </TableCell>
      </TableRow>
      {expandedId === exp.experiment_id && (
        <TableRow className="bg-background/50">
          <TableCell colSpan={7} className="p-0">
            <ExperimentDetail experiment={exp} />
          </TableCell>
        </TableRow>
      )}
    </Fragment>
    ```

    Add `Fragment` import: `import { useState, Fragment } from "react";`

    **Issue 2 -- Hypothesis truncation (EXPR-02 readability portion):**
    Remove the truncation logic completely. The current code at lines 97-102:
    ```tsx
    <span className="truncate max-w-[300px] block">
      {exp.hypothesis.length > 60
        ? exp.hypothesis.slice(0, 60) + "..."
        : exp.hypothesis}
    </span>
    ```
    Replace with just: `{exp.hypothesis}` (no wrapping span, no slice, no truncate class).

    Additionally, add `className="whitespace-normal"` to the Hypothesis `<TableCell>` to override the default `whitespace-nowrap` from the table.tsx base component. This allows long hypothesis text to wrap naturally within the cell rather than forcing the table to expand horizontally.

    Keep the existing column width classes on the header (`w-12` for #, `w-[100px]` for 2023 Val Acc, `w-20` for 2022 Val/2021 Val/Log Loss/Status) -- these constrain the fixed-width columns and let the Hypothesis column absorb remaining space naturally via the `w-full` table.
